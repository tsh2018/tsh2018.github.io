单例设计模式

单例：一个类在内存中只有一个对象，别人不能再去创建对象
1.必须在该类中创建一个对象、
2.私有化自己的构造器，防止外界通过构造器来创建新的对象
3.对外界提供一个方法，能够获取已经创建好的对象

抽象类与接口

抽象方法必须在抽象类或者抽象接口中
即加 abstract, 且抽象方法必须被子类去覆盖此方法，抽象方法也不能是private，static和final
用了final表示不能被覆盖
抽象方法是没有方法体的

抽象类必须有子类，一般被当作父类被继承
抽象类也不能创建对象
抽象类中可以有抽象方法， 也可以有非抽象方法
抽象类必须有子类才有意义，即有子类继承
子类没有去覆盖抽象方法，可以把子类也变成抽象方法，让孙子类再去覆盖抽象方法
抽象类中的构造方法不能私有化
抽象类名字前面通常有Abstract
不能有fina修饰l

接口定义

interface 接口名称{}
接口是一个特殊的抽象类
如果接口内的方法没有加上abstract，编译时自动会加上 void trans(); –> public abstract void trans();接口中的方法也可以没有方法体（有abstract）
接口中默认成员变量 public static final
接口不能创建对象
接口中定义的变量全部都是全局静态常量
接口中定义的方法是公共抽象方法，不能有普通方法
接口中没有构造器
接口中可以定义内部类，也是Public abstract
接口是可以继承的，且可以多继承， 在extends
后面的接口以逗号隔开，类不能多继承

类实现接口（接口也可以做父类）

class 类的名称（extents classname）implements 接口的名称{}
类实现方法必须是public来修饰（覆盖的时候，子类的方法权限要比父类大，或者一样大，不能比他小）

面向接口编程

把实现类对象赋值给接口类型变量
即多态的好处，屏蔽了不同类之间的实现差异，从而达到通用编程

内部类（在类中的一个类）

1.增强封装，不允许其他类访问这个内部类
2.内部类可以访问外部类中的成员

外部类的修饰符只能有两个 public或默认修饰
内部类可以使用很多个修饰符

创建实例内部类（前面没有什么修饰的类）

有static修饰表示是类中的成员，可以直接访问

静态内部类（在内部类前面加上static）

静态内部类中没有外部类的引用，但是可以访问外部类的静态成员
访问静态内部类的静态成员可以不用创建对象直接通过 “ . ”调用 如Outter.Inner.color，不需要创建外部类对象

局部内部类（定义在方法中的内部类）

不能使用一些修饰 public private。。。
只能在定义的方法中使用
局部内部类不能包含静态变量
局部内部类可以包含局部变量，使用局部变量的本质是final 常量（在方法区中）

匿名内部类（没有名字的局部内部类）

匿名内部类必须要有父类或者实现接口之后才能使用
没有构造器
只使用一次的时候用匿名内部类
new 父类构造器 或 接口（）{
内部写的代码，在new的时候会自动执行
}
